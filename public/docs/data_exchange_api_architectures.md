Q1. What is the primary difference between standard HTTP REST APIs and WebSockets?

A. REST APIs support JSON, WebSockets do not.
B. REST is stateless request-response; WebSockets provide persistent bidirectional stateful connections.
C. WebSockets only run on local servers.
D. REST APIs require HTTPS whereas WebSockets are completely unencrypted.

Answer:

✅ B. REST is stateless request-response; WebSockets provide persistent bidirectional stateful connections.

Explanation:

HTTP REST follows a request-response pattern. WebSockets open a single persistent TCP connection allowing real-time push events from both sides.

---

Q2. Which HTTP method is idempotent and designed to update an existing resource or create it if missing?

A. POST
B. GET
C. PUT
D. DELETE

Answer:

✅ C. PUT

Explanation:

PUT is idempotent, meaning multiple identical requests yield the same system state.

Q3. What is JSON in API exchanges?

A. Java Socket Open Network
B. JavaScript Object Notation - a lightweight text format for structured data.
C. Joint Server Output Node
D. A binary protocol for image compressions.

Answer:

✅ B. JavaScript Object Notation - a lightweight text format for structured data.

Explanation:

JSON (JavaScript Object Notation) is the standard lightweight format used to transmit structured data in modern APIs.
