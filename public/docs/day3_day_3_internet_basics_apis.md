### API Selection Cheat Sheet

| Scenario | Best Choice |
| :--- | :--- |
| CRUD Application | REST |

Q1. What is the Internet?

A. Google
B. A web browser
C. A network of networks
D. A data center

Answer:

✅ C. A network of networks

Explanation:

The Internet connects millions of networks worldwide, allowing devices to communicate.

---

Q2. What is the primary purpose of a Router?

A. Store files
B. Route network traffic between networks
C. Create websites
D. Generate AI responses

Answer:

✅ B. Route network traffic between networks

Explanation:

Routers decide the best path for data packets to travel.

Q10. What is the main job of DNS?

A. Store videos
B. Convert domain names into IP addresses
C. Create websites
D. Send emails

Answer:

✅ B. Convert domain names into IP addresses

Explanation:

DNS translates human-readable domain names like google.com to computer-routable IP addresses.

Q21. What problem does GraphQL solve?

A. Hardware acceleration
B. Requesting exactly the data needed
C. GPU memory
D. DNS lookup

Answer:

✅ B. Requesting exactly the data needed

Explanation:

GraphQL prevents over-fetching and under-fetching by letting the client ask for exactly what it needs.

---

Q22. When should GraphQL be preferred?

B. For hardware drivers
C. For DNS management
D. For SSD storage

Answer:

✅ A. When frontend needs flexible data retrieval

GraphQL gives frontends high flexibility in retrieving nested and tailored data fields.

---

Q23. What is a major advantage of gRPC?

A. Human readability
B. Faster machine-to-machine communication
C. Browser rendering
D. DNS resolution

Answer:

✅ B. Faster machine-to-machine communication

Explanation:

gRPC uses Protocol Buffers and HTTP/2 to enable fast, low-overhead binary serialization.

---

Q24. Which environment commonly uses gRPC?

Q25. SOAP is best known for:

B. Strict standards and enterprise integrations

Answer:

✅ B. Strict standards and enterprise integrations

Q26. What makes WebSockets different from REST?

Explanation:

WebSockets support full-duplex communication over a single, long-lived TCP connection.
